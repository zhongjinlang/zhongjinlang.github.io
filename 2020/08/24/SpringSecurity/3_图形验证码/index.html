三：图形验证码

发表于 2020-08-24 | 分类于 SpringSecurity | | 阅读数
字数统计: 1.1k | 阅读时长 ≈ 5m

验证码为了防止恶意用户暴力重试而设置的，不管是注册、登入，如不加以限制，一旦某些恶意用户利用计算机发起无限重试，就很容易使系统遭受破坏

3.1 过滤器方式实现

在Spring Security中，实现验证码校验方式有多种。最简单的方式就是自定义一个专门处理验证码逻辑的过滤器，将其添加到 Spring Security过滤器链的合适位置。当匹配到登入请求时，立刻对验证码进行校验，如果成功则放行， 否则提前结束整个验证请求

引入开源的验证码组件

配置kaptcha实例

创建一个获取图形验证码Controller

有了验证码API，接下来自定义验证码校验过滤器，只需继承Filter即可，在spring体系中，推荐使用OncePerRequestFilter实现，它可保证一次请求只会通过一次该过滤器(filter实际不能保证这一点)

定义失败时候处理的方法以及验证码失败的异常处理

3.1.2 OncePerRequestFilter

自定义验证码过滤器，实现逻辑并不复杂，验证码仅仅核对用户提交的验证码以及session中保持的验证码是否一致

3.1.3 addFilterBefore

将过滤器添加到Spring Security的过滤器链中即可生效
